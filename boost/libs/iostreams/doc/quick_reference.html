Core Components

These components form the interface between Boost.Iostreams and the standard iostreams library.

Name Header[1] Description
stream_facade stream_facade.hpp Stream template which performs i/o using an instance of its first template parameter, which must model the concept Device. Derives from std::basic_istream, std::basic_ostream or std::basic_iostream depending on whether its first template parameter models Source, Sink or both.
streambuf_facade streambuf_facade.hpp Stream template which performs i/o using an instance of its first template parameter, which must model the concept Device.
filtering_stream filtering_stream.hpp Stream template which performs filtered i/o using a contained filter chain. Derives from std::basic_istream, std::basic_ostream or std::basic_iostream, depending on its template parameters.
filtering_streambuf filtering_streambuf.hpp Stream buffer template which performs filtered i/o using a contained basic_chain.
basic_chain chain.hpp Sequence of zero or more Filters, followed by an optional Device, accessed with a stack-like interface. Used by filtering_stream and filtering_streambuf.
code_converter code_converter.hpp Device adapter which takes a narrow-character Device and produces a wide-character Device by introducing a layer of code conversion.

Views

Each view consists of one or more class templates together with an object generator.

View Template Object Generator Header[1] Description
combination combine combine.hpp Takes a pair of Filters or Devices and yields a filter or device which performs input using the first component and output using the second.
composite compose compose.hpp Takes a Filter and either a second Filter or a Device and yields a Filter or Device whose output flows first through the first component and then through the second and whose input flows in the reverse direction.
inverse invert invert.hpp Takes a Filter and yields a Filter that performs the same filtering algorithm but is an OutputFilter if the given component is an InputFilter, and an InputFilter otherwise.
restriction restrict restrict.hpp Takes a Filter or Device together with a stream offset and an optional length and yields a Filter or Device for accessing the specifed subquence of the given component
tee_device tee tee.hpp Takes a pair of Sinks and yields a Sink which sends all incoming data to both of the given Sinks.
tee_filter Takes a Filter and a Sink and yields a Filter which sends all outgoing data to the Sink in addition to passing it downsteam.
