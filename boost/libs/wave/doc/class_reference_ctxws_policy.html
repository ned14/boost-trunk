Introduction

This context policy is used to define the whitespace handling behaviour to be implemented by the iterators generated by a instance of the boost::wave::context<> template.

This policy type is used as a template parameter to the boost::wave::context<> object, where the default policy provides functionality to skip all whitespace or all whitespace except comments.

Header wave/whitespace_handling.hpp synopsis

namespace boost {
namespace wave {
namespace context_policies {
 
    template <typename Token>
    struct eat_whitespace 
    {
        // test, whether a given token may be skipped
        bool may_skip (TokenT &token, bool &skipped_newline);
    };

}}}   // namespace boost::wave::context_policies

Template parameters

Every whitespace handling policy should expect to be instantiated with the token type as a template parameter.

Member functions

may_skip

    bool may_skip(TokenT &token, bool& skipped_newline);

The function may_skip will be called by the library, whenever a token is about to be returned to the calling application.

The token parameter hold a reference to the current token. The policy is free to change this token if needed.

The skipped_newline parameter holds a reference to a boolean value which should be set to true by the policy component whenever a newline is going to be skipped.

If the return value is true, the given token is skipped and the preprocessing continues to the next token. If the return value is false, the given token is returned to the calling application. Caution has to be used, because by returning true the policy is able to force skipping even significant tokens not only whitespace.

Example

The following code snippet shows a simple example how to implement a whitespace handling policy preserving all whitespace in the generated output:

    template <typename TokenT>
    struct eat_whitespace 
    {
        // not any of the given tokens may be skipped anytime
        bool may_skip (TokenT &token, bool &skipped_newline)
        { return false; }
    };
