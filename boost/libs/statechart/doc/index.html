Overview

Welcome to boost::fsm, a C++ library for finite state machines. Features include:

  • Straightforward transformation from UML state chart to executable C++ code and vice versa (no code generator required)
  • Comprehensive UML semantics support (history not yet implemented)
    • Hierarchical (composite, nested) states
    • Orthogonal (concurrent) states
    • Entry-, exit- and transition-actions
    • Guards
  • Error handling support
  • Full type-safety
  • State-local storage
  • Customizable resource acquisition

The library is not yet complete. Here is the current to-do list:

  1. The library does not yet check for invalid transitions between different orthogonal regions, although the documentation claims otherwise.
  2. Factor out several functions in the state_machine class template into base classes to reduce code size in projects with many different state machines.
  3. Add a concurrent_state_machine class template to model a state machine running in its own thread.
  4. Add regression tests.
  5. Port to gcc.
  6. Add reference documentation.
  7. Add support for shallow/deep history.

This is a preliminary submission to encourage feedback. All comments are most welcome!
