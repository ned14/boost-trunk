Consequently, template metaprograms tend to be syntactically obtuse, and often compile quite slowly. The latter occurs where the parsers use algorithms designed for "small N" with bad asymptotic performance, say the number of instances of a template, and the metaprograms cause N to be quite big. The designers of C++ parsers never intended for them to be exploited as language interpreters. It's not clear if the template metaprogramming market is big enough to encourage parser vendors to improve the situation.
