compose semantics

(from correspondence, very raw)

compose_f_gx<F1,F2>::apply<T>::type is an equivalent to F1::apply<F2::apply<T>::type>::type. To clarify things further, below is (an incomplete) table of compose_xxx templates semantics as I see it: 

compose_f_gx<F1,F2>::apply<T>::type == F1::apply<F2::apply<T>::type>::type
compose_f_gx_hx<F1,F2,F3>::apply<T>::type == F1::apply<F2::apply<T>::type,F3::apply<T>::type>::type
compose_f_gx_hy<F1,F2,F3>::apply<T1,T2>::type == F1::apply<F2::apply<T1>::type,F3::apply<T2>::type>::type
compose_f_gxy<F1,F2>::apply<T1,T2>::type == F1::apply<F2::apply<T1,T2>::type>::type
compose_f_x_hy<F1,F2>::apply<T1,T2>::type == F1::apply<T1,F2::apply<T2>::type>::type
compose_f_gx_y<F1,F2>::apply<T1,T2>::type == F1::apply<F2::apply<T1>::type,T2>::type
compose_f_x_x<F1>::::apply<T>::type == F1::apply<T,T>::type
