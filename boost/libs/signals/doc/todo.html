Boost.Signals: TODO

The following list contains a list of changes and enchancements that are planned for the Boost.Signals library.

  • Factor the <multimap> include out of public headers using the pImpl idiom. This should reduce compile times and object file sizes.
  • Support for tracking weak_ptrs (need to poll and lock weak_ptrs when calling).
  • Introduce a connection proxy that allows connections/disconnections to a signal but does not allow invocation. (idea by Thomas Witt)
  • Clarify the role of a const signal.
  • Thread-safe implementation.
