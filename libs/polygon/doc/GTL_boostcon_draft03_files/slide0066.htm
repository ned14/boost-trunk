What code that uses GTL looks like
•Two lines of code in the example invoke five different GTL algorithms
•Arguments passed into functions are not GTL data types
•The code is maximally concise, yet easy to read
•Clip b to the bounding box of a, XOR that with a shrunk by ten then merge into result
•Details of memory management for intermediate results are abstracted away from the use of algorithms
•Such code is easy to write and easy to maintain
void foo(list<CPolygon>& result,
         const list<CPolygon>& a,
         const list<CPolygon>& b) {
  CBoundingBox domainExtent;
  gtl::extents(domainExtent, a);
  result += (b & domainExtent) ^ (a - 10);
}
